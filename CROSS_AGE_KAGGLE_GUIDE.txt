======================================================================================
CROSS-AGE GENERALIZATION: STEP-BY-STEP GUIDE FOR KAGGLE
======================================================================================

OBJECTIVE: Train on adult speech, test on children's speech to analyze age robustness

TASKS:
1. Train model using adult speech samples and test on children's speech
2. Analyze how accent cues differ across age groups  
3. Compare robustness of MFCC vs HuBERT for age generalization

======================================================================================
STEP 1: PREPARE DATA LOCALLY
======================================================================================

1.1. Create child audio metadata
-----------------------------
Run locally in your project directory:

```bash
python create_child_metadata.py
```

This will:
- Scan data/raw/child_audios/ for audio files
- Detect state/accent from filename or folder structure
- Create data/raw/child_audios/child_metadata.csv

IMPORTANT: Review the generated CSV and fix any incorrect state labels!

Expected output:
  ✓ Created metadata for X child audio files
  ✓ Saved to: data/raw/child_audios/child_metadata.csv
  Distribution by state: [shows counts per state]

1.2. Update Kaggle package
--------------------------
Create updated package with child audio files:

```powershell
# Create new zip with child audios included
Compress-Archive -Path data, src, scripts, *.py, requirements.txt `
                 -DestinationPath kaggle_cross_age_package.zip -Force
```

This will be ~3.5-4GB (includes child_audios)

======================================================================================
STEP 6: INTERPRET RESULTS
======================================================================================

6.1. Key Metrics to Look At
---------------------------

A. ACCURACY COMPARISON
   Adult Acc (Train) vs Child Acc (Test)
   - Shows how well model generalizes to different age group
   - Lower child accuracy = poor age generalization

B. GENERALIZATION GAP
   Gap = Adult Acc - Child Acc
   - Smaller gap = better age robustness
   - MFCC vs HuBERT: which has smaller gap?

C. CONFUSION MATRICES
   - Which accents are confused more in children?
   - Are errors consistent across feature types?

6.2. Expected Findings
---------------------

Hypothesis 1: MFCC may generalize better
   Reason: Handcrafted features capture age-invariant characteristics
   - MFCC statistics less sensitive to pitch changes
   - Formant ratios remain stable across ages

Hypothesis 2: HuBERT may struggle more
   Reason: Learned features may overfit to adult voice characteristics
   - Higher pitch in children's voices
   - Different speaking rates
   - Shorter utterances

Hypothesis 3: Both show accuracy drop
   Reason: Fundamental acoustic differences between ages
   - Children: higher F0, different formant frequencies
   - Different pronunciation patterns
   - Less stable articulation

6.3. Analysis Questions
----------------------

1. Which feature type has smaller generalization gap?
   → More robust to age differences

2. Which accents maintain accuracy on children?
   → These have age-invariant characteristics

3. Which accents drop significantly?
   → Age-dependent pronunciation variations

4. Are confusion patterns similar across features?
   → Shared vs feature-specific challenges

======================================================================================
TROUBLESHOOTING
======================================================================================

Issue 1: "FileNotFoundError: child_metadata.csv"
Solution: Ensure child_metadata.csv was created and included in zip

Issue 2: "No child audio files found"
Solution: Check folder structure in zip matches expected paths

Issue 3: GPU memory error
Solution: Reduce batch_size in train_model function (32 → 16)

Issue 4: All child predictions are wrong
Solution: Check if state labels in child_metadata.csv are correct

Issue 5: Very low accuracy on both
Solution: Verify audio files are not corrupted, check sample rate
